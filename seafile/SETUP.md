# Инструкция по установке Seafile (для админа)

## Шаг 1: Настройка сервера

### 1.1. Установить Docker
```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
```

### 1.2. Создать папку и скачать конфиг
```bash
mkdir -p /opt/seafile
cd /opt/seafile
wget https://raw.githubusercontent.com/haiwen/seafile-docker/master/docker-compose.yml
```

### 1.3. Отредактировать docker-compose.yml
Замените все `ПРИДУМАЙТЕ_ПАРОЛЬ` на свои пароли (в 3 местах):
- MYSQL_ROOT_PASSWORD
- MYSQL_PASSWORD  
- DB_ROOT_PASSWD
- SEAFILE_ADMIN_PASSWORD

Также замените:
- `SEAFILE_SERVER_HOSTNAME=seafile.yourcompany.com` на адрес вашего сервера

### 1.4. Запустить
```bash
docker-compose up -d
```

### 1.5. Подождать 3-5 минут пока запустится
```bash
docker logs -f seafile
```

Когда увидите "Seafile services started" — готово.

---

## Шаг 2: Настройка общего хранилища

### 2.1. Откройте в браузере
`http://IP_ВАШЕГО_СЕРВЕРА`

Логин: admin@yourcompany.com  
Пароль: тот что указали в SEAFILE_ADMIN_PASSWORD

### 2.2. Создайте библиотеку
- Нажмите "+" → "Новая библиотека"
- Название: `Проект`
- Тип: Личная (потом расшарите)

### 2.3. Создайте внутреннюю ссылку для синхронизации
- Откройте библиотеку "Проект"
- Нажмите "Поделиться" → "Внутренняя ссылка"
- Скопируйте ссылку (формат: `https://seafile.example.com/lib/XXXX-XXXX-XXXX-XXXX/sync`)

---

## Шаг 3: Настройка пользователей (800 чел)

### 3.1. Создайте тестовую библиотеку для скрипта
- Создайте библиотеку "Синхронизация"
- Внутри создайте текстовый файл с ссылкой синхронизации
- Скопируйте ссылку

### 3.2. Создайте пользователей
Можно через админку или CSV-загрузку.
